git: submodule 的使用

Pro git

在项目开发中，可能会遇到：某个工作中的项目需要包含并使用另一个项目。 也许是第三方库，或者你独立开发的，用于多个父项目的库。

可以将库的代码拷贝到当前项目，但是当库上游进行修改时，任何你进行的定制化的修改都很难归并。

git submodule 通过子模块处理这个问题。子模块允许你将一个 Git 仓库当作另外一个Git仓库的子目录。这允许你克隆另外一个仓库到你的项目中并且保持你的提交相对独立。
